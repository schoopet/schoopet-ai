# Changelog — 2026-06-01

## Discord Approval Flow

- **Disable approval buttons immediately** — buttons are disabled as soon as a user clicks one, preventing double-approvals.
- **Group Discord approval prompts** — multiple pending approvals are batched into a single Discord message instead of sending one per resource.
- **Ack Discord interactions before agent lookup** — gateway acknowledges the Discord interaction within the 3-second window before starting the (potentially slow) agent call, eliminating timeout errors.

## Agent Tools — Drive / Docs / Sheets

- **Normalize Drive resource URLs** — all Drive/Docs/Sheets tools now accept either a bare file ID or any Google URL form as input and always return canonical `https://docs.google.com/…` URLs. Fixes resource-confirmation cache poisoning when URL forms differ across calls.
- **Trim sheet/doc tool surface; swap `fetch_url` for ADK `url_context`** — removed redundant helper tools, replaced the custom web-fetch tool with the built-in ADK `url_context` mechanism.
- **Rich Google Docs formatting** — new `append_formatted_to_doc` tool writes headings, bold/italic text, bullet lists, and tables to a Doc using the Docs API Batch Update. Existing `write_to_doc` now supports overwrite mode.
- **Sheet column-header support** — sheet read/write tools understand named column headers.
- **Prompt guidance for both agents** — added formatting and `url_context` usage notes to root agent and deep research agent system prompts.

## Deep Research Agent

- **Flatten deep_research_agent** — replaced the `LoopAgent` wrapper with a direct single-`LlmAgent` implementation, simplifying the control flow and reducing overhead.
- **Expand toolset for autonomous offline runs** — deep research agent now has access to `create_google_doc`, `create_spreadsheet`, Drive search, async task tools, and code execution so it can save results without human interaction.

## Async Tasks

- **Align gateway Cloud Tasks creation** — extracted Cloud Tasks HTTP client into `sms-gateway/src/internal/cloud_tasks.py`; task payloads and headers are now consistent between the gateway and the task worker.
- **Track retries and progress** — async task documents now record `attempt_count`, `last_error`, and a `progress` field updated by the agent during long-running work.
- **Fix async research rescheduling** — corrected a bug where a rescheduled deep-research task could re-trigger resource confirmation instead of resuming execution.

## Agent Engine / AgentEngineClient

- **Compatibility with `google-cloud-aiplatform` 1.154.0** — `AgentEngineClient` updated to use the new async class-method API (`async_create_session`, `async_delete_session`, `stream_query`) introduced in aiplatform 1.154.0.
- **Sessions IAM role** — added `aiplatform.reasoningEngines.sessions.*` permissions to the `agentEngineUser` custom IAM role so the gateway service account can manage sessions.

## Agent Core

- **Fix `GlobalGemini` GC closure bug** — `api_client` is now a `cached_property` instead of a plain property, preventing the underlying `httpx` transport from being garbage-collected between calls.
- **Explicit `app_name` on `AdkApp`** — prevents the Agent Engine resource ID from being used as the agent display name after deployment.
- **Prompt and parser fixes from prod session audit** — corrected calendar tool date-range handling, Drive tool confirmation edge cases, and time-tool timezone formatting.

## Dependencies

- **Upgrade `google-adk` to 2.1.0** — picks up session streaming fixes and eval improvements; minor version bumps for `google-cloud-aiplatform`, `google-generativeai`, and gateway dependencies.

## Infrastructure / CI

- **Dev environment engine ID** — `PERSONAL_AGENT_ENGINE_ID` set in `environments/dev.env` for the dev Agent Engine instance.
- **Agent deploy workflow** — added `workflow_dispatch` trigger and `environments/**` path filter to the GitHub Actions deploy-agent workflow.
- **Dev agent renamed** — dev agent display name changed to `BooniBear-Dev`; prod name set to `BooniBear`.
- **Dev Terraform fix** — added `sms_gateway_url` to `dev.tfvars` to fix OIDC audience mismatch that caused 503s on the dev gateway.
- **IAM Connector provisioning script** — `scripts/provision_iam_connector.sh` added; `apis.tf` enables the IAM Connector API for OAuth 3LO support.
